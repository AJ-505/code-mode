Abstract: Evaluative Benchmarking of Code Mode vs. Traditional Function-Calling in Agentic Workflows

As agentic AI systems become more capable, the dominant interaction pattern remains conventional function-calling, in which models are guided through predefined tool schemas and structured invocation formats. While effective for many tasks, this pattern may impose avoidable overhead in complex workflows by expanding prompt context, constraining action selection, and increasing brittleness across multi-step reasoning loops. An alternative now gaining attention is code mode, where the model is allowed to generate executable code against a bounded interface rather than repeatedly selecting from static function definitions. This shift raises an important research question: can code mode improve the efficiency and reliability of agentic systems without sacrificing control, safety, or evaluability?

This study proposes a benchmark framework for comparing code mode against traditional function-calling across realistic agent workflows. The goal is not merely to compare outputs, but to measure how each interaction protocol shapes agent behavior under equivalent objectives and constraints. The benchmark will evaluate both paradigms on matched tasks using standardized interfaces, structured output requirements, and consistent scoring criteria. Particular attention will be given to token economy, latency, completion quality, and robustness under multi-step task execution. Where possible, the study will emphasize controlled conditions so that differences in performance can be attributed to the protocol itself rather than unrelated environmental variance.

The expected contribution is a practical and reproducible methodology for studying protocol choice in agent design. By framing code mode not simply as a product feature but as an experimental alternative to schema-heavy tool invocation, this work aims to clarify when executable-code interaction offers a meaningful advantage, where it introduces new risks, and how future agent architectures might balance flexibility with operational discipline.
